# FSC22 Benchmark: Frozen Audio Embeddings

Code for the paper:

> **Label-Efficient Forest Sound Classification with Frozen Audio Foundation Model Embeddings**

---

## Overview

This repository contains the code to reproduce all experiments in the paper. The benchmark compares three families of forest sound classifiers on the [FSC22 dataset](https://www.kaggle.com/datasets/irmiot22/fsc22-dataset):

- **Frozen audio foundation model embeddings** — seven pretrained encoders (CLAP, AST, BEATs, PANNs CNN14, YAMNet, VGGish, OpenL3) with shallow classifier heads
- **Handcrafted acoustic features** — MFCC statistics with SVM; log-mel statistics with XGBoost
- **ImageNet-pretrained CNN backbones** — ResNet-18, EfficientNet-B0, MobileNetV2, DenseNet-121 trained end-to-end on log-mel spectrograms

All three arms are evaluated under identical splits, seeds, and protocol across five repetitions with paired significance tests.

## Requirements

```bash
pip install -r requirements.txt
```

BEATs is not pip-installable. Clone the source separately:

```bash
git clone https://github.com/microsoft/unilm.git /path/to/unilm
```

The BEATs extractor adds the correct path automatically if you set the `UNILM_DIR` environment variable:

```bash
export UNILM_DIR=/path/to/unilm
```

## Data

Download FSC22 from Kaggle and place the audio files and metadata under `data/`:

```
data/
  FSC22/          # audio clips (.wav)
  metadata.csv    # class labels
```

Alternatively, set environment variables to point to a custom location:

```bash
export FSC22_AUDIO_DIR=/path/to/FSC22
export FSC22_METADATA=/path/to/metadata.csv
```

## Reproducing the experiments

Run all phases in order:

```bash
python reproduce.py
```

Or run individual phases:

```bash
python reproduce.py --phase extraction       # extract embeddings (GPU recommended)
python reproduce.py --phase embeddings       # frozen embedding experiments
python reproduce.py --phase handcrafted      # handcrafted feature experiments
python reproduce.py --phase cnn              # CNN baseline training (GPU required)
python reproduce.py --phase lowdata          # data-efficiency analysis
python reproduce.py --phase augmentation     # augmentation fairness check
python reproduce.py --phase efficiency       # efficiency measurements
```

Results are written to `results/`. Figures and tables are generated by:

```bash
python analysis/generate_figures.py
python analysis/generate_tables.py
python analysis/statistical_tests.py
```

## Repository structure

```
config.py                   central configuration (paths, hyperparameters, seeds)
reproduce.py                master script to run all phases
extraction/
  extract_all.py            extract embeddings from all seven encoders
  extractors/               one file per encoder + handcrafted features
experiments/
  run_embedding_clf.py      frozen embedding + classifier experiments
  run_handcrafted.py        handcrafted feature experiments
  run_cnn_baseline.py       CNN baseline training
  run_augmented_cnn.py      augmentation fairness check (CNN side)
  run_augmented_embedding.py augmentation fairness check (CLAP side)
  measure_efficiency.py     extraction and inference timing
analysis/
  generate_figures.py       all paper figures
  generate_tables.py        all paper tables
  generate_table1_dataset.py  Table 1: FSC22 class distribution
  generate_table8_lowdata.py  Table 8: low-data regime results
  statistical_tests.py      paired t-tests and Benjamini-Hochberg correction
```
